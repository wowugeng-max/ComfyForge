一、项目概述
1.1 背景
ComfyUI 作为强大的 AI 工作流引擎，已被广泛用于图像、视频生成。然而，随着创作深度增加，用户面临三大痛点：
•  
资产无法复用与溯源：精心调试的提示词、角色形象、工作流无法系统化管理，每次创作都要重新“造轮子” 
 
。
查看来源详情。打开侧边栏。
•  
缺乏前置创意编排：剧本、分镜的策划与最终的渲染割裂，缺乏一个统一的“画布”将其串联 
 
。
查看来源详情。打开侧边栏。
•  
执行后端僵化：无法在同一个工作流中，灵活地将大语言模型（提词/编剧）与本地高算力 GPU（渲染）以及云端 API 进行无缝的混合调度 
 
。
查看来源详情。打开侧边栏。
1.2 项目目标
ComfyForge 旨在打造一个非侵入式、以“无限画布”为核心的智能创作中枢，实现：
•  
资产即服务 (AaaS)：将提示词、图片、工作流模板等统一为资产。支持双轨制（项目专属 vs 全局公共）隔离，支持拖拽式复用 
 
。
查看来源详情。打开侧边栏。
•  
大一统多模态流：彻底打通本地 ComfyUI、云端算力与大语言模型（Gemini/Qwen 等）的界限，在单一画布内实现数据穿透 
 
。
查看来源详情。打开侧边栏。
•  
极致心流交互：采用 Figma/Notion 级的现代化布局，硬核复刻纯正的节点编辑器交互（双击搜索、免鼠标流、完美橡皮筋缩放） 
 
。
查看来源详情。打开侧边栏。
1.3 核心理念
•  
非侵入式：不修改 ComfyUI 本身，通过 API 与 WebSocket 控制，与官方生态保持兼容 
 
。
查看来源详情。打开侧边栏。
•  
沙盒隔离：以“项目”为绝对物理容器，确保创作数据的纯净与聚合 
 
。
查看来源详情。打开侧边栏。
•  
数据驱动：前后端极度解耦，UI 面板基于后台模型配置按需生成（Schema-Driven） 
 
。
查看来源详情。打开侧边栏。

三、核心功能与使用场景
3.1 黄金链路场景
•  
场景一（前置策划）：拖入“分镜大师”提示词资产 -> 连线大模型节点 -> 输入剧本生成极简英文分镜描述 -> 存为资产 
 
。
查看来源详情。打开侧边栏。
•  
场景二（图生图演排）：拖入左侧图片资产与工作流资产 -> 连线 ComfyUI 物理节点 -> 本地 GPU 满载渲染 -> 返回高精度成图 
 
。
查看来源详情。打开侧边栏。
•  
场景三（视频合成）：聚合多张分镜图资产 -> 连线视频大模型节点 -> 渲染动态短片 
 
。
查看来源详情。打开侧边栏。
