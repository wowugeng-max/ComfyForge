一、项目概述
1.1 背景
ComfyUI 作为强大的 AI 工作流引擎，已被广泛用于图像、视频生成。然而，用户面临两大痛点：

资产无法复用，每次创作都要重新“造轮子”。

执行后端僵化，只能依赖本地 GPU，无法灵活选择云端 API 或云 ComfyUI 实例。

ComfyForge 旨在打造一个 ComfyUI 的智能驾驶舱，实现资产化管理和灵活调度，并引入智能学习系统，让推荐规则越用越聪明。

1.2 项目目标
资产即服务：统一管理提示词、角色、工作流模板、生成物等，支持版本、标签、血缘追溯。

执行即调度：支持本地 ComfyUI、云端 ComfyUI、直接 API 三种执行模式，自动路由最优后端。

智能推荐：根据用户历史配置，自动推荐常用参数，并可配置规则和阈值。

双模式创作：同时提供“引导式执行”（新手友好）和“无限画布”（专家自由）两种创作方式，覆盖全层次用户。

1.3 核心理念
非侵入式：不修改 ComfyUI 本身，通过 API 控制。

资产驱动：一切可沉淀的灵感都是资产，让创作过程可积累。

灵活扩展：适配器模式，易接入新 API 或云平台。

学习进化：系统通过用户行为不断优化推荐，越用越智能。

二、当前项目状态（截至 2026.3.4）
✅ 已完成功能 (近期重大突破)
1. 基础框架与资产管理

后端 FastAPI + SQLite 稳定运行，前端 React + React Flow + Zustand 架构成型。

资产 (Asset) CRUD API 完善，支持类型过滤、血缘追踪。

2. 核心调度与模型管理 (🔥 最新增强)

手动模型注入：突破官方 API 限制，支持手动添加隐藏/内测模型（如 Veo 3.1 视频生成），并在前端抽屉中与官方同步模型隔离管理。

健康追踪探针系统：模型具备 health_status（可用/额度干了/无权限等）。支持前端“一键单测”，并在画布节点中以“红绿灯”直观显示模型状态，避免无效调用烧毁额度。

底层架构支持多 Key 绑定、动态工厂模式 (Adapter Factory) 异步调度。

3. 无限画布引擎 (🔥 核心闭环跑通)

节点注册与渲染：实现了 BaseNode，成功注册 LoadAssetNode (加载资产)、GenerateNode (AI生成器) 和 DisplayNode (展示器)。

动态参数面板：GenerateNode 会根据模型能力 (文本/识图/绘图/视频) 自动切换 UI 模式，并过滤可用模型。

物理连线与数据流转 (Data Flow)：彻底打通了 React Flow 的边 (Edges) 与节点数据 (Nodes Data) 的联动：

逆向溯源 (Pull)：GenerateNode 运行时自动顺着左侧连线，吸取上游 LoadAssetNode 中的图片/文本资产，打包进 incoming_assets 发送给后端。

正向分发 (Push)：生成成功后，自动顺着右侧连线，将生成的 res.data 注入到下游的 DisplayNode 中。

多模态渲染展示：DisplayNode 能够根据接收到的结果类型（text, image, video），自适应渲染 Markdown 文本框、可放大预览的图片或视频播放器。
