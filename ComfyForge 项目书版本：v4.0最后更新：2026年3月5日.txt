ComfyForge 项目书版本：v4.0最后更新：2026年3月5日作者：与你一起讨论的伙伴一、项目概述1.1 背景ComfyUI 作为强大的 AI 工作流引擎，已被广泛用于图像、视频生成。然而，随着创作深度增加，用户面临三大痛点：资产无法复用与溯源：精心调试的提示词、角色形象、工作流无法系统化管理，每次创作都要重新“造轮子”。缺乏前置创意编排：剧本、分镜的策划与最终的渲染割裂，缺乏一个统一的“画布”将其串联。执行后端僵化：无法在同一个工作流中，灵活地将大语言模型（提词/编剧）与本地高算力 GPU（渲染）以及云端 API 进行无缝的混合调度。1.2 项目目标ComfyForge 旨在打造一个非侵入式、以“无限画布”为核心的智能创作中枢，实现：资产即服务 (AaaS)：将提示词、图片、工作流模板等统一为资产。支持双轨制（项目专属 vs 全局公共）隔离，支持拖拽式复用。大一统多模态流：彻底打通本地 ComfyUI、云端算力与大语言模型（Gemini/Qwen 等）的界限，在单一画布内实现数据穿透。极致心流交互：采用 Figma/Notion 级的现代化布局，硬核复刻纯正的节点编辑器交互（双击搜索、免鼠标流）。1.3 核心理念非侵入式：不修改 ComfyUI 本身，通过 API 与 WebSocket 控制，与官方生态保持兼容。沙盒隔离：以“项目”为绝对物理容器，确保创作数据的纯净与聚合。数据驱动：前后端极度解耦，UI 面板基于后台模型配置按需生成（Schema-Driven）。二、当前项目状态（截至2026.3.5）✅ 已完成功能Phase 1-3：基础底座与 API 管道后端引擎大一统：全面引入 @ProviderRegistry 插件注册表，统一了 OpenAI, Gemini, Qwen 等大模型的调用协议。完善的 Key 管理模块与统一生成网关（/generate）。Phase 6：前端大重构（现代化中枢）彻底废弃旧版 Gradio 界面，采用 React 19 + Vite + Ant Design 6 重构。实现了卡片式项目流大厅（Dashboard），支持项目的 CRUD 与封面预览。Phase 7：无限画布生态 (Infinite Canvas 2.0)基于 React Flow 实现了无限画布，并利用 JSON 序列化实现了画布状态的永久固化（读档/存档）。双击召唤魔法：关闭原生缩放，实现了“双击画布空白处 -> 弹出悬浮菜单 -> 自动聚焦搜索 -> 回车建节点”的专业级心流交互。智能多模态节点生态：大脑节点 (GenerateNode)：支持文本/图/视频四模态无缝切换，动态生成参数面板，支持外部 System Prompt 资产注入。展示节点 (DisplayNode)：多模态结果渲染，一键将生成物固化为项目资产。资产输入节点 (LoadAssetNode)：拖拽承载，具备“变色龙端口”（文本绿、图片蓝、工作流紫）防呆连线设计。Phase 8：双轨制资产管理 (Dual-Track Asset System)数据库扩展 canvas_data 与 project_id 字段。侧边栏支持一键切换 [ 📦 项目专属 ] 与 [ 🌍 全局公共 ] 作用域，完美解决隔离与复用的冲突。🚧 进行中/待完善功能ComfyUI 物理引擎节点 (ComfyUIEngineNode)：接收工作流 JSON 与大模型 Prompt，下发至本地 5090 算力渲染。WebSocket 实时通信：捕获本地/云端 ComfyUI 的 K-Sampler 进度条与实时预览图。三、核心功能与使用场景3.1 黄金链路场景场景一（前置策划）：拖入“分镜大师”提示词资产 -> 连线大模型节点 -> 输入剧本生成极简英文分镜描述 -> 存为资产。场景二（图生图演排）：拖入左侧图片资产与工作流资产 -> 连线 ComfyUI 物理节点 -> 本地 GPU 满载渲染 -> 返回高精度成图。场景三（视频合成）：聚合多张分镜图资产 -> 连线视频大模型节点（Luma/Wan2.2）-> 渲染动态短片。3.2 资产管理功能支持类型：提示词 (prompt)、图像 (image)、视频 (video)、工作流模板 (workflow)。隔离机制：基于 project_id 的沙盒隔离，以及无 ID 的全局公共库。拖拽空投：基于 react-dnd 实现侧边栏到画布的精准资产投放。四、技术架构4.1 当前实现架构Plaintext┌─────────────────────────────────────────────────────────────┐
