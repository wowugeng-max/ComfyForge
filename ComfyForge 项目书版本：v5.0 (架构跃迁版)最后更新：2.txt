ComfyForge 项目书版本：v5.0 (架构跃迁版)最后更新：2026年3月8日状态：前端 UI/UX 大一统完成，即将开启后端“万物皆配置”架构重构一、项目概述1.1 背景ComfyUI 作为强大的 AI 工作流引擎，已被广泛用于图像、视频生成。然而，随着创作深度增加，用户面临三大痛点：资产无法复用与溯源：精心调试的提示词、角色形象、工作流无法系统化管理，每次创作都要重新“造轮子”。缺乏前置创意编排：剧本、分镜的策划与最终的渲染割裂，缺乏一个统一的“画布”将其串联。执行后端僵化：无法在同一个工作流中，灵活地将大语言模型（提词/编剧）与本地高算力 GPU（渲染）以及云端 API 进行无缝的混合调度。1.2 项目目标ComfyForge 旨在打造一个非侵入式、以“无限画布”为核心的智能创作中枢，实现：资产即服务 (AaaS)：打造具有现代商业软件质感（DAM）的资产大厅，支持双轨制（项目专属 vs 全局公共）隔离，支持拖拽式复用。大一统多模态流：彻底打破文本、图像、视频生成的边界，在同一个画布上完成“剧本 -> 分镜 -> 视频”的全链路闭环。万物皆配置 (Config-Driven Engine)：抛弃臃肿的硬编码适配器，通过底层统一的多模态 Payload 协议和 Jinja 模板映射，实现新 AI 模型的“零代码接入”。二、核心架构 (v5.0 升级版)系统采用前后端分离架构，注重模块化与动态配置。2.1 前端 (React 19 + Vite + React Flow)无限画布引擎：基于 Zustand 状态管理，实现了完美的“液态自适应缩放”节点矩阵。支持丝滑的撤销/重做、静默防抖保存策略（实时/定时/手动）。商业级 UI 视效：全面采用明亮、清爽、充满呼吸感的现代浅色风格。支持毛玻璃悬浮操作、Grid 瀑布流卡片、智能参数映射抽屉。动态表单渲染：基于后端 Schema，动态生成模型配置与参数表单。2.2 后端 (FastAPI + SQLite)API 底座：提供 RESTful API，承载项目、资产、算力凭证、模型的 CRUD。多模态通用代理引擎 (Universal Proxy Adapter)：[v5.0 核心] 废弃按厂商硬编码的插件模式，引入基于 JSON/YAML 配置的路由引擎。通过统一内部协议（Unified Payload）和 Jinja 模板渲染，实现对全网大模型 API（OpenAI 兼容及自定义格式）的动态适配。双轨调度器：DirectAPI: 面向大语言模型与云端 API 的极速直连。LocalComfy: 面向本地物理机/远端集群的 ComfyUI WebSocket 长连接调度。三、核心功能矩阵 (当前已就绪形态)3.1 无限画布枢纽 (Canvas Workspace)液态节点矩阵：大脑节点、展示节点、资产节点、算力引擎节点全部实现无视内部媒体固有尺寸的“绝对百分比响应式缩放”。极客级心流：支持画布双击全局搜索节点、一键抽屉式资产库拖拽加载（Drag & Drop）。安全护航：具备完整历史状态栈（Ctrl+Z / Ctrl+Y）及无打扰后台静默固化机制。3.2 数字资产大厅 (Asset Hub)瀑布流视效：针对图像、视频、提示词、工作流等提供沉浸式的多模态预览卡片，配合鼠标悬浮毛玻璃操作栏。顶级过滤舱：支持基于 Segmented 控制器的项目沙盒隔离、模态类型极速筛选。双栏铸造车间：新建与编辑资产采用专业级“双栏属性检查器”布局，JSON 代码框极客感拉满。3.3 算力与模型中枢 (Compute & Models)双子星管理体系：融合 <Tabs> 设计，左侧管理凭证（Key Vault），右侧管理模型（Model Matrix）。凭证保险箱：凭证密钥全面采用密码学遮罩，本地安全落库。集成一键连通性 Ping 测试。全脑自动感应：新增凭证后，后端自动触发全量模型拉取落库；同时也支持从侧边宽屏抽屉中“手动注入”未知模型。参数级渗透：支持为单个模型注入高阶参数（Max Tokens, Temperature 等），完美适配 ModelParamEditor。四、实施路线图（Roadmap）阶段目标核心任务状态Phase 0环境搭建初始化项目，配置 FastAPI + SQLite✅ 完成Phase 1-3API管道与底座DirectAPI执行器，数据库表设计，Key与模型同步✅ 完成Phase 4-5前端基建与通信ReactFlow 接入，FastAPI 跨域通信，多模态输出机制✅ 完成Phase 6-7AaaS
