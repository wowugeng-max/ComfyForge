一、项目概述
1.1 背景
ComfyUI 作为强大的 AI 工作流引擎，已被广泛用于图像、视频生成。然而，用户面临两大痛点：

资产无法复用，每次创作都要重新“造轮子”。

执行后端僵化，只能依赖本地 GPU，无法灵活选择云端 API 或云 ComfyUI 实例。

ComfyForge 旨在打造一个 ComfyUI 的智能驾驶舱，实现资产化管理和灵活调度，并引入智能学习系统，让推荐规则越用越聪明。

1.2 项目目标
资产即服务：统一管理提示词、角色、工作流模板、生成物等，支持版本、标签、血缘追溯。

执行即调度：支持本地 ComfyUI、云端 ComfyUI、直接 API 三种执行模式，自动路由最优后端。

智能推荐：根据用户历史配置，自动推荐常用参数，并可配置规则和阈值。

双模式创作：同时提供“引导式执行”（新手友好）和“无限画布”（专家自由）两种创作方式，覆盖全层次用户。

1.3 核心理念
非侵入式：不修改 ComfyUI 本身，通过 API 控制。

资产驱动：一切可沉淀的灵感都是资产，让创作过程可积累。

灵活扩展：适配器模式，易接入新 API 或云平台。

学习进化：系统通过用户行为不断优化推荐，越用越智能。

二、当前项目状态（截至 2026.3.3）
✅ 已完成功能
基础框架

后端：FastAPI + SQLite，支持异步任务、数据库初始化。

前端：React + TypeScript + Vite，Ant Design 组件库，React Router 路由，Zustand 状态管理。

资产管理

资产模型（Asset）支持 prompt、image、character、workflow、video 等类型。

资产 CRUD API，支持按项目过滤、版本控制、血缘追踪。

前端资产列表、详情、创建/编辑表单（动态类型切换）。

Key 管理

API Key 模型（APIKey），支持多提供商、额度、优先级、标签。

Key 管理 API（增删改查、测试、批量测试）。

前端 Key 管理页面，支持添加、编辑、测试、删除。

智能路由引擎（初步实现）。

执行器

直接 API 管道执行器（DirectAPIPipelineExecutor），支持多步骤、变量替换、资产引用。

本地 ComfyUI 执行器（LocalComfyExecutor），支持工作流提交、轮询、文件下载。

视频循环执行器（RealVideoLoopExecutor），支持多段视频生成（手动指定首帧、尾帧、提示词资产）。

云端视频循环执行器（CloudVideoLoopExecutor，待完善）。

工作流配置器（核心亮点）

上传 JSON 工作流文件，React Flow 可视化渲染节点图。

节点点击打开参数配置面板，支持自定义参数名称。

批量配置面板：上传后自动弹出，汇总所有推荐字段，支持全选、自定义、批量保存。

节点 ID 快速定位。

已配置参数列表展示，支持定位和删除。

只读模式与编辑模式区分。

“刷新建议”按钮，可手动重新触发推荐面板。

智能学习系统

统计模型 NodeParameterStat，记录节点类型下各字段被配置的次数。

推荐接口 /suggestions/recommend 返回热门字段。

上报接口 /suggestions/report 收集用户配置。

前端 workflowSuggestions.ts 整合本地规则和统计推荐，实现冷启动和学习进化。

规则管理系统

规则模型 RecommendationRule，存储手动规则（节点类型、字段、默认参数名、优先级、阈值等）。

规则 CRUD API。

合并接口 /recommendation-rules/combined 返回手动规则和学习规则。

前端规则管理页面，展示手动和学习规则，手动规则可增删改查，学习规则可一键转为手动规则。

规则支持优先级排序、阈值控制（仅当统计次数≥阈值时自动勾选）。

无限画布（Phase 7.1 完成）

基于 React Flow 的节点编辑器基础框架。

节点类型注册机制（nodeRegistry.ts），支持动态添加自定义节点。

画布状态管理（canvasStore.ts），实现节点/边的添加、删除、移动、连接。

撤销/重做功能（Zustand + immer）。

节点库侧边栏（可拖拽放置节点）。

基础节点组件 BaseNode，统一样式和端口。

示例节点 LoadAssetNode（基础版）和 GenerateNode（基础版）已注册。

三、技术架构
3.1 后端技术栈
框架：FastAPI

数据库：SQLAlchemy + SQLite

核心模块：

api/：资产、项目、Key、规则、推荐、执行等路由

core/：执行器、资产工具、Key 监控、路由引擎

models/：数据库模型（资产、Key、统计、规则等）

adapters/：第三方 API 适配器（Qwen、Gemini 等）

3.2 前端技术栈
框架：React 18 + TypeScript

构建工具：Vite

UI 库：Ant Design 5

路由：React Router 6

状态管理：Zustand

可视化：React Flow

拖拽：react-dnd + react-dnd-html5-backend

HTTP 客户端：axios

3.3 核心数据流
text
用户上传工作流 JSON → workflowToFlow 转换为 React Flow 节点/边 → 显示画布
→ 点击节点 → 打开参数面板（结合规则和统计推荐）
→ 用户勾选并命名 → 保存到 parameters 状态
→ 保存资产 → 上报统计 → 更新推荐数据库
